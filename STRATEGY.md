# 炎炎消防队(Yanyan)Strategy

## Target problem

已经出现鼻炎、血糖偏高或慢性炎症的人,已经意识到要改变饮食,但**不知道从哪下手**。信息太多太矛盾(生酮 / 16:8 / 低 GI / 各种说法),搜出来的内容一半在卖货、一半在贩卖焦虑;医生只说"少吃糖",但不会告诉他第二天早饭该吃什么。

## Our approach

押**「中医语言的火分 + 30 天发物档案」双交付**:用户每餐拍照,产品用「发 / 温和 / 平」(中医发物语言)给红黄绿;每日首屏中心是「今日体质:中火,火分 62」(上火等级主、数字辅);次日早晨 30 秒打卡(鼻塞 / 起痘 / 口干 / 大便 / 精神 / 浮肿 / 喉咙痒);累计满 14 天起渐进显示个性化发物候选清单,30 天整合为可分享的「发物档案」PDF。

**主交付**(日活锚)= 火分:用户每天打开看的是上火等级 + 火分波动 + 归因 breakdown(饮食 / 环境 / 作息)。这是 Levels / Veri / Whoop 这条赛道的玩法,我们用中医词汇做差异化 UI。
**辅交付**(留存锚)= 发物档案:30 天后用户拿到一份属于自己、可累积、可分享给妈妈/医生的 PDF。这是发物清单的最终形态,不与火分互推,服务不同的信任层。

诚实承认这是"用中医词汇加持的 Whoop/Veri",不是另一个赛道——差异化由两点支撑:**(1)中医发物 / 上火 / 寒热语言的母语级共鸣**(西方产品做不出);**(2)拍照零硬件**(对手都需要 CGM / 手环)。会主动得罪减肥/卡路里人群 + 西方循证科学权威叙事人群,把用户筛到"已体检异常 + 听过妈妈讲发物"的中产。

核心赌注:中医诊断 2000 年的"望闻问切→食物寒热温凉"逻辑从未被产品数据化过;Levels / Veri / Whoop 不针对中餐;Flammy / DII 类在中文圈无渗透;西方话语权和中文认知体系之间有一个无人占据的真空。

## Who it's for

**Primary:** 30-45 岁、体检报告里有 1-2 项异常(血糖偏高、尿酸高、或常年鼻炎)、不愿意吃药的中产 —— 他们雇 Yanyan 来:**(1)** 在每餐做选择的瞬间得到一个"发不发"的明确答复;**(2)** 在 14-30 天后拿到一份只属于自己的"发物清单",从此知道哪几样食物对**自己这个体质**最容易引发次日反应。因为他们现在只能在体检后焦虑半年、然后忘掉,等下次体检再焦虑一次,也只能听妈妈泛泛地说"少吃发物"却不知道自己的发物到底是哪几样。

## Key metrics

- **周活跃记录率(WAR)** — 过去 7 天里至少有 5 天每天上传 ≥2 餐 + 完成次晨打卡的用户占比。会跌的指标,反映"双闭环(食物拍照 + 次晨体感)有没有真的在用"。来源:产品事件埋点。
- **次晨打卡完成率** — 当日有食物记录的用户中,次日早晨 30 秒打卡完成的占比。这是 24 小时反馈闭环能不能撑起来的命门。来源:产品事件埋点。
- **个体发物清单生成率** — 注册满 14 天的用户中,系统已为其回归出 ≥3 项个体发物的占比。这是产品核心资产是否真正交付的硬指标——清单不出来,产品没价值。来源:产品数据库。
- **季度异常项改善率** — 主动上传过两次体检报告的用户中,至少 1 项异常指标(空腹血糖 / 尿酸 / CRP / 鼻炎发作频次)改善的占比。来源:用户上传 + OCR 解析。
- **减肥目的用户占比 < 30%** — 新用户问卷"主要目标=减肥"的占比。反向指标:超过阈值说明定位漂了、获客在错误人群上烧钱。来源:新用户引导问卷。

## Tracks

### 食物→发物分类的本土权威性

本土食物库 + 中医发物 / 寒热温凉 / 上火分类引擎 + 中医专家审核机制(老中医 KOL 池、发物分类共识线下专家会、典籍引用映射)。同时维护"中医分类 ↔ 西方营养(DII / AGEs / GI)"的双层映射,后台保留科学版本以备出海或精英用户切换。

_Why it serves the approach:_ 这是产品的认知护城河——西方抗炎话术在中文圈是认知外来语,中医发物是母语。只有母语级的分类才能在中国 30-45 岁中产家庭里产生即时共鸣。

### 拍照 + 次晨体感的双闭环

每餐拍照 → 红/黄/绿(发/温和/平) + 当下"下一餐避免什么"。次日早晨 30 秒打卡(鼻塞/起痘/口干/大便/精神/浮肿/喉咙痒)+ 昨日饮食回归。**接入手机健康 App 的步数、心率、血氧、睡眠**作为体感打卡的客观补丁。

_Why it serves the approach:_ 拍照是输入,次晨打卡是验证——两者一起才是中医的"食物-身体"完整闭环。任意单边都不能产生"你的发物清单"这个核心交付物。

### 个体发物回归引擎

7-14 天滚动数据回归 + 中医群体先验作为 Bayesian prior + 个体数据做 update,在小样本下输出"对你而言概率最高的 3-5 样发物"。30 天复盘报告 + 季度对照中医辨证术语翻译给用户。

_Why it serves the approach:_ 个性化发物清单是产品的核心资产,也是 LTV 的根基——它不可被竞品复制、有迁移成本、可被用户自豪地分享给家人和医生。

### 精准获客与反向定位

把"已体检异常 + 已经听过妈妈/外婆讲发物概念"的中文用户捞进来,把减肥人群推出去。投放素材语言全部用"发物 / 上火 / 寒热"而非"卡路里 / 减脂",引导问卷、产品文案三处都要执行这一筛选。

_Why it serves the approach:_ 这是反向指标(减肥用户 <30%)的抓手,也是定位不漂的保险丝。语言筛选本身就是用户筛选——用"发物"而不是"炎症",会自动筛掉看不懂的、留下天然认同的。

## Not working on

- **公开向中国大陆开放(v1 阶段)** — v1 数据存储在 Supabase 境外 region,中国大陆用户健康信息出境在《个保法》第 38 条下需 CAC 安全评估。v1 仅做私人 beta(邀请制 / 海外华人 / 自测),验证 PMF 后(预计 6 个月)迁阿里云华东 region 再公开。
- **健身教练 / 私教课程** — 能赚钱,但偏离"已体检异常中产"的核心场景。
- **保健品 / 电商** — 最容易掉进去的变现路径,会把工具变成卖货平台、毁信任。
- **儿童 / 老人 / 孕妇等特殊人群** — 医学风险高、获客模式完全不同,先不碰。
- **西方"抗炎科学权威"叙事** — 这条路被 Levels / Veri / ZOE 占着,我们刻意让出。需要 hs-CRP / DII 严格循证语言的精英用户/海外华人,我们暂不服务(后台保留映射,但不打这个招牌)。
- **保险/医疗精算变现路径** — 与中医叙事的循证标准存在结构性冲突,Phase 1 不碰;若未来要做,需另起科学版产品线。
